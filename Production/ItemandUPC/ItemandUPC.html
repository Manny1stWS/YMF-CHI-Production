STYLES AND UPC NUMBERS

Prepared by Manuel Herrera

Updated: 1/12/24

How to Create Style Numbers:

In the Production Server go to “Style & UPC” Folder located in: Z:\Production\Style & UPC 09-30-22.

Open the “Style # 2011” excel file. Do not make changes to the file if you open it as Read-Only because someone else has the file opened and is creating Styles. Send a message through the Production Team Chat to close the file if they don’t need it.

Style numbers are set up by the customer and normally begin with a 2-letter prefix made up from the customer’s name and another letter representing the category of the Style.

Each customer will have its own tab where style numbers can be created for the corresponding customer if the styles you are creating will not use Royalty Brands Packaging. All Styles that will use Royalty Brands Packaging must be entered in the YMF tab.

In column “A” you will enter the customer’s prefix plus the category letter. For example, if the customer is Dollar General and the Style’s category is Accent Rug, you will enter DGA.

Do not use "O" for other if the category of the Style you are creating is not listed here. Ask management to assign a letter to the category you are working on.

In column "B" there will be a 6-digit code setup from 000001 to 001000 to start. If necessary the number can be increased up to 999999.

In column "C" there is a formula that combines column A & B once you enter a description in column "D".

Descriptions in column "D" must be made in the following order:

  1. For the 1st part of the description, use one of these or a combination these: Brand Name, Style's Category, Collection Name, Product Unique Characteristic, and Design's Name:
    1. Brand Name - normally used in the packaging.
    2. Style's Category - Always use this as the first word specially if you are bringing assortments of multiple designs.
    3. Collection Name - this is normally dictated by the person who gave you the order.
    4. Design' Name - Only use this as the first word in the description if you are ordering a design that has multiple sizes in multiple colors.
  2. For the 2nd part of the description, use the size(s):
    1. When an Style has multiple sizes to make a set, write the biggest size followed by the number of sizes in the set. For example, if you are ordering a 20x30, 20x20 and 20x19 as a set, you will write "20x30 3pc Set". If you are ordering 2pc Sets, write the 2 sizes. For example, "18x30/17x24 2pc Set".
  3. For the 3rd part of the description, use the design's name:
    1. If the design's name is not used in the first part of the description, use it on the 3rd part.
    2. If the Style you created has multiple designs, you can write "Assorted".
  4. For the 4th part of the description, write the main color of the Style.:
    1. If the Style (single sku) has multiple colors but there is a color that is predominant, write that color.
    2. If the Style (single sku) has multiple colors but it is difficult to tell which color is predominant, write "Multi".
    3. If the Style (single sku) only has 2 colors, write the 2 colors.
    4. If the Style (multiple skus) with multiple colors, write "Assorted".

Please note that in Exenta there are 2 style description limits:

  1. The long form description “Style Desc” has a 100 characters limit.
  2. The short form description “Style Name” has 30 characters limit. If you need to reduce the number of characters, start by abbreviating the word “Assorted” if used. Or reduce the number of sizes listed if the Style is a set; For example, only write “2pc Set”. If this is not enough, abbreviate color’s name.

These are other rules you should follow for the descriptions to keep things clean and consistent in Exenta:

  1. Do not write the description with all capital letters.
  2. Do not write the description with all small letters.
  3. Write the first letter of every word in the description capitalized.
  4. Make sure there is only one space between words.
  5. When you paste descriptions into Navision, remove any special characters that come up.
  6. Make sure the description does not start with extra spaces.
  7. Keep in mind that the sales department want to be able to sort the Styles by description to see a group of Styles of the same kind together.

Write your name's initials next to the description and make sure you save the excel file before closing it.

If you have a new customer which is not listed in any of tabs, ask management to add it.

How to assign UPCs to Style Numbers:

In the Production Server go to “Style & UPC” Folder located in: Z:\Production\Style & UPC 09-30-22

Open the “UPC Check Digit 2021-06-24” excel file. Do not make changes to the file if you open it as Read-Only because someone else has the file opened and is assigning UPCs. Send a message through the Production Team Chat to close the file if they don’t need it.

Please note that not all Style#s need UPC. Only assign UPCs to the Style#s that need them. Customers like Ross and Burlington require pre-ticketing with the customer's UPCs. Do not assign UPCs to these types of Styles.

Make sure there are no filters set in the excel file. If you see the symbol below on any of the top headings in the excel file, it means there is a filter.

You can remove all the filters in the excel by going to Data on the top section of the excel and clicking Clear:

If you leave the filter and paste a group of Styles and descriptions, you might be pasting over other Style#s and descriptions already in the file. Make sure you do not paste over Style#s and description already in the file.

The UPC Check Digit will have a list of all the UPCs assigned or unassigned in column "X". Assigned UPCs will have Style#s, Description, Issued by and Prepack in columns Y, Z, AA and AB.

Find a UPC that does not have an Style# and Description in columns Y and Z.

Copy the Style# and description (Columns C and D) you created in Style & UPC and do a paste values using this symbol after right clicking the cell you will paste. Enter your initials in columns AA and the case pack in column AB:

Make sure you save before closing the file.
